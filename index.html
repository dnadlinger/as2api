as2api

About

This is a JavaDoc-a-like for ActionScript 2.0.

It began it because there don't seem to be any complete, free (GPL or simmilar), API documentation tools, as of November 2004.

The scripts are implemented in Ruby, and work under Windows, Mac OSX, and Linux. A GUI is available for Windows (and soon, OSX too).

Status

Starting to be useful. Still to be done:

  • Proper command-line options
  • Have some way of pre-populating namespace with standard types (String, Object) so that type resolution doesn't complain when they're not found
  • Hide private class members
  • Automatically pull any package.html contents into package-summary page
  • Be package/class centric, rather than sourcepath centric (i.e. we should be given a list of packages to document, and just load what's required to document the API of those types)
  • Automatically include the default CSS with the generated files (if there's no file there already)

Examples

com.xfactorstudio.xml.xpath
Documentation for the XFactor Studio XPath API.
com.asunit.framework
Documentation for the AsUnit unit-testing framework.

Usage

Windows All-in-One Executable

On extracting the zip file, you should be left with as2api_win32.exe and style.css. On running the executable, you should see a window like that shown.

  1. Use the Add... button to add entries to the classpath list. These entries must be the folders that contain your ActionScript packages (i.e. not the folders that contain the .as files themelves).
  2. Use the Browse... button to choose where the generated files should be placed. (NB there is currently no button to create a new folder, although there needs to be one. Sorry.)
  3. Press Generate Docs... and wait for processing to finish
  4. Copy style.css into the output directory by hand

Ruby Scripts

To make use of the plain script files, you must have a copy of Ruby installed. The distribution archive contains a number of files, with the main script being documentor.rb. Run this script, from the directory that contains it, by saying:

dave@castle:~/projects/as2api$ ruby documenter.rb path/to/sources

The script will create a directory called apidoc within the current directory. You place a copy of style.css into this directory by hand.

Current

Windows all-in-one GUI .exe: as2api-allinone-w32-0.2.zip ~1.8MiB
Plain Ruby scripts as2api-0.2.tar.gz ~11KiB

Changes since 0.1:

  • GUI for Windows
  • Frameset navigation pages now generated
  • Method and field indexes contain links to methods and fields inherited from super-types
  • Support get and set implicit property access functions
  • Notice constructor functions as such, and present differently in documenation
  • Sort lists alphabetically, in various spots
  • Tiny Ruby 1.6 compatability fix
  • Hack to derive package-name from source path, when no package is specified in the source file
  • Don't give error for intrinsic keyword
  • Don't treat is as a keyword
  • Ignore [attributes] preceeding types and type-members, rather than producing a parse error
  • Generate documentation for fields as well as methods
  • Ignore #import keyword (allow parsing to continue, rather than cause a parse error)
  • Fix known causes of parse errors due to missing semicolons
